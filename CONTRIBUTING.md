# Contributing

Thanks for your interest in improving Órbita. This document explains how to propose changes and what to expect from the review process.

## How to contribute

1. Fork the repository and create a feature branch from `main`.
2. Make sure `pnpm lint`, `pnpm test:run` and `pnpm build` pass.
3. Open a pull request against `main`.
4. Address review feedback and keep the PR updated with the base branch.

## Expectations

- Keep changes small and focused.
- Update documentation when behavior or setup instructions change.
- Follow the existing code style and test patterns.

## Security

Please do not report security vulnerabilities through public GitHub issues. See `SECURITY.md` for the responsible disclosure process.
